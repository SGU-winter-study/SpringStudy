## Section 6 스프링 DB 접근기술

스프링은 다양한 데이터 베이스(로컬 : H2, 로컬 X: MariaDB, MyBatis, HikariCP)와 
여러 데이터 액세스 기술(?) (Native JDBC, JdbcTemplate, JPA)를 이용해 DB를 접근하고
관리한다.

### OCP(개방-폐쇄원칙) 
스프링의 DB기술은 기존 코드의 수정없이 DB와 관련된 코드와 SpringConfig만 수정하는 
것을 통해 구현 클래스를 변경할 수 있다

### 스프링의 DB관리

1. Native Jdbc : 말그대로 직접 모든 sql문과 코드를 작성하는 방식
2. Jdbc Template : Native Jdbc에서 발전된 형식으로 반복코드를 제거해준다.
3. JPA : 반복코드 제거는 물론, SQL작성도 자동으로 해준다. 이외에도 다양한 장점이 있다.

* 스프링 데이터 JPA 
: JPA를 좀 더 편리하게 작성할 수 있게 도와주는 기술, 자동으로 CRUD 단순조회 기능 등을 지원해준다.

* Querydsl
: 데이터 JPA로 작성하기 어려운 동적쿼리 작성에 이용되는 라이브러리, 하지만 이 라이브러리로도
모든 SQL을 커버할 순 없기 때문에 백엔드 엔지니어로써 SQL 지식이 필요하다.

## Section 7 AOP 적용
AOP를 개념적으로는 이해했지만, 내부로직이 이해 안가서 구글링을 통해 좀 더 이해하는 것이 필요해 보임.

- AOP(Aspect Oriented 프로그래밍)
: 관점을 바탕으로 메소드, 기능을 나누어 보고 이를 프로그래밍에 응용하는 방식?
  예를 들어 공통 관심 사항(공통으로 필요한 기능)과 핵심 관심 사항(핵심적인 기능)을 
  나누어서 프로그래밍 하는 방식을 의미한다.

- 스프링의 AOP
: 스프링에서는 프록시 패턴을 이용한 AOP를 적용하고 있다.
  원리는 타겟 메소드 실행전에 프록시로 wrapping 시켜놓아서 프록시를 먼저 실행한후 메소드가 
  실행되게 하는 방식이다.(@Before 어노테이션과 유사한듯?)
